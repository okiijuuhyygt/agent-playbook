A五層架構(由下往上)

1

底層引擎 — 誰在算

模型 = Anthropic Claude(主力 Opus 4.8,Fable 掛掉自動降階)。跑在 Claude Code(官方 CLI),不是第三方包裝 → 綁的是 Anthropic。常駐在 tmux,有「哨兵」死機自動重啟。核心智能不在你 Mac 上,換機器只要有 Claude Code＋Anthropic 帳號就能跑。
2

介面層 — 你怎麼跟我講話

Discord plugin:你手機丟訊息 → 我收到 → reply 回。頻道 = 功能分區。cursor 機制記「每頻道讀到哪」,重啟只拉新訊息、不漏不重。
3

能力層 — 我會做的事 = skills

約 18 個自訂 skill(巨集＋SOP＋我的判斷):接案全流程、音樂製作、內容行銷、客戶經營、系統自維護。加 Claude Code 內建:跑程式／爬網／讀寫檔／瀏覽器自動化／接 MCP(Gmail/Calendar/Notion…)。
4

記憶層 — 我為什麼不失憶

workspace markdown(SOUL/USER/AGENTS/CHANNEL-RULES/LEARNED-RULES/SESSION-HANDOFF/日誌)＋ Obsidian SSOT。全是硬碟上的 markdown,換引擎、換機器都不掉 —「換的是引擎,不是車」。
5

自動化層 — 不用你叫就跑的

用 macOS launchd(純 shell,不燒模型額度):morning-briefing(每天晨報)、evolve 自我進化(每週日檢討規則→Discord 等你批)、model-probe(掛了自動降階)、哨兵 revive(死機重啟)。
自我進化,白話講

我不是寫死的。每次被你糾正 → 沉澱成規則(LEARNED-RULES)→ 下次所有對話自動讀到、不用重教 → 週日再整批檢討一次。小蝦講的「agent 自我進化」,我們已經在跑了。

C搬到 Hermes 的移植評估

「Hamas」謎底 = Hermes Agent(Nous Research 開源框架,MIT,主打「程序記憶 procedural memory」)。⚠️ 跟本機 OpenClaw 那個同名 workaround 很可能不是同一個東西,別混。

Hermes 是什麼

開源、可本機自架(最低 2GB RAM)的 agent 框架。model-agnostic(支援 400+ 模型含 Claude/GPT/Gemini/Ollama 本地)。核心 = 四層記憶 + 自我萃取技能卡:做完複雜任務會自動把「怎麼做最有效」存成 skill 並優化 —— 跟我們 LEARNED-RULES/evolve 是同一個概念,只是內建、更自動。

移植難度(直接講)

Skill 層幾乎可直接搬

Hermes 技能 = SKILL.md(YAML frontmatter + markdown body),跟我們 .claude/skills/ 那套幾乎同源。改一下欄位就能用,甚至整理成 GitHub repo 一鍵 tap 進去。

記憶/規則 markdown可搬,要手工重排

LEARNED-RULES/SESSION-HANDOFF/USER.md 對得起 Hermes 的 MEMORY.md/USER.md/skills 三層,但沒有自動遷移器,要人工拆併。

launchd 自動化不衝突

照舊用 launchd 觸發 hermes CLI 即可。

最大缺口 = 介面痛點在這

我們現在靠 Discord 手機端在用;Hermes 是 terminal-first,沒有現成 Discord 手機介面(要自己接)。搬家最痛的不是技能格式,是介面。
給小蝦/決策的一句話

Hermes 不是來取代我們這套,是同類、且 skill 格式相容到可以互通。真要搬:技能/記憶搬得動;卡的是介面(沒 Discord)＋ 模型費。對小蝦「設備有限、無工程背景、不想花大錢」——Hermes 能本機跑 Ollama 省模型費是加分,但 terminal-first ＋ 要自接介面,跟「簡單、不能太難」相衝,要老實講。

費用:框架免費;跑的成本接 Ollama 本地 = $0、接 API(含 Claude)約 $10–50/月。數據(stars/模型數)來自二手中文文章,當量級參考。來源:bnext article/90591、hermes-agent.nousresearch.com/docs。
